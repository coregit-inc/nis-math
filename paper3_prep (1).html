Q1 · Sorting

// 2025: bubble sort descending
// 5 marks · 10 min

2025 Question

verbatim

"A teacher records the scores of 20 maths tests. (a) Suggest a suitable data structure to store student scores. (b) Write a code or pseudocode to sort the student scores in descending order using bubble sort."

Answer (a) — 1 mark

1D array (also accepted: list, vector).

Answer (b) — 4 marks

FOR i ← 0 TO 18 DO          // 1 mark: outer loop
    FOR j ← 0 TO 18-i DO      // 1 mark: inner loop with valid range
        IF arr[j] < arr[j+1] THEN  // 1 mark: comparison (DESCENDING uses <)
            t ← arr[j]
            arr[j] ← arr[j+1]         // 1 mark: swap with temp variable
            arr[j+1] ← t
        ENDIF
    ENDFOR
ENDFOR
Common errors: using > instead of < for descending · forgetting the temp variable · wrong loop bounds (off-by-one).

Bubble sort — both directions (memorise both)

core

Ascending (smallest first)

FOR i ← 0 TO n-2 DO
    FOR j ← 0 TO n-2-i DO
        IF arr[j] > arr[j+1] THEN
            // swap
            t ← arr[j]
            arr[j] ← arr[j+1]
            arr[j+1] ← t
        ENDIF
    ENDFOR
ENDFOR

Descending (largest first)

FOR i ← 0 TO n-2 DO
    FOR j ← 0 TO n-2-i DO
        IF arr[j] < arr[j+1] THEN
            // swap
            t ← arr[j]
            arr[j] ← arr[j+1]
            arr[j+1] ← t
        ENDIF
    ENDFOR
ENDFOR

Only difference: the comparison operator (> vs <).

2026 Forecast — Insertion Sort. The syllabus (11.5.2.4) lists "sort by insert and bubble". Bubble was tested in 2025; insertion sort is overdue. Memorise it now:

Insertion Sort (forecast)

2026 likely

Idea

Build a sorted prefix one element at a time. For each new element, shift larger values right until it lands in the correct spot.

FOR i ← 1 TO n-1 DO
    key ← arr[i]
    j ← i - 1
    WHILE j >= 0 AND arr[j] > key DO
        arr[j+1] ← arr[j]   // shift right
        j ← j - 1
    ENDWHILE
    arr[j+1] ← key      // insert at correct slot
ENDFOR

Trace [5, 2, 8, 1] ascending

StepArrayComment
start[5, 2, 8, 1]—
i=1, key=2[2, 5, 8, 1]shift 5 right, insert 2
i=2, key=8[2, 5, 8, 1]8 > 5, no shift
i=3, key=1[1, 2, 5, 8]shift 8, 5, 2 right, insert 1

Complexity: O(n²) worst · O(n) best (already sorted). Stable sort.

Analysis

  • (b) Condition is on line 3 (BRP 06)
  • (c) Loop body lines: 1, 2, 3, 7, 8
  • (d) Purpose: computes m[08] mod m[09] — i.e. the remainder of dividing 6 by 4 = 2

How to trace systematically

technique
  1. Set up columns: Address · AC · each memory address used · Condition · Output
  2. Initialise the DAT values in the first row (don't run any instruction yet)
  3. For each instruction, copy the previous row's values, then update only what changes
  4. For branches, write TRUE/FALSE in the Condition column
  5. Track the address column carefully — branches change the next instruction
Mark scheme: awards 1 mark per column correctly filled. Even if AC values are wrong, you get marks for filling memory and condition columns correctly.

2025 — Binary Search Pseudocode (completed)

model answer
INPUT name
low ← 0
index ← -1
high ← LENGTH(PhoneBook) - 1
flag ← FALSE

WHILE (low <= high OR flag <> TRUE)        // blank #1
    mid ← INT((low + high) / 2)
    IF PhoneBook[mid] = name THEN          // blank #2
        index ← mid                            // blank #3
        flag ← TRUE                             // blank #4
    ELSE IF PhoneBook[mid] > name THEN
        high ← mid - 1                          // blank #5
    ELSE
        low ← mid + 1
    ENDIF
ENDWHILE

IF flag = TRUE THEN                          // blank #6
    OUTPUT("Found at index:", index)
ELSE
    OUTPUT("Not in array")
ENDIF

Complexity (1 mark each)

  • Time complexity: O(log n) — also accepted O(log 100)
  • Worst-case iterations for n=100: 7 (because ⌈log₂ 100⌉ = 7)

How binary search works

step by step

Sorted array. Repeatedly check the middle. If too small → search right half. If too big → search left half. Halves the search space each iteration.

Trace example: search for 23 in [3, 8, 12, 17, 23, 29, 35, 41]

Iterlowhighmidarr[mid]Action
10731717 < 23 → low = 4
24752929 > 23 → high = 4
344423match — return 4 ✓

3 iterations for n=8. Worst case ≈ ⌈log₂ 8⌉ = 3.

2026 Forecast. Could be: linear search instead of binary (different complexity O(n)), OR a binary search variant (find first occurrence, find insertion point). Could also ask to convert recursive ↔ iterative.

Linear Search (forecast — easier alternative)

2026 backup
FUNCTION linearSearch(arr, target)
    FOR i ← 0 TO LENGTH(arr) - 1 DO
        IF arr[i] = target THEN
            RETURN i
        ENDIF
    ENDFOR
    RETURN -1
ENDFUNCTION

Complexity: O(n) worst case · O(1) best case (target at index 0). Works on unsorted arrays.

Stack vs Queue (memorise)

core
Stack — LIFO

Last In First Out. Add and remove from the same end (top). Operations: push(x), pop(), peek(), isEmpty(), isFull(). Use cases: function call stack, undo, browser back.

Queue — FIFO

First In First Out. Add at rear, remove from front. Operations: enqueue(x), dequeue(), front(), isEmpty(), isFull(). Use cases: print spool, task scheduling, BFS.

2026 Forecast — Queue. Syllabus 12.5.2.1 says "stack and queue". Stack appeared in 2025; queue is overdue. Same kind of question (transform State A → State B), but with enqueue/dequeue.

Queue Operations (forecast)

2026 likely

Front pointer on the left. New items enqueue at the rear. Dequeue removes from the front.

// Initial queue: [A, B, C]   front→A, rear→C

enqueue("D")    // [A, B, C, D]
dequeue()       // removes A → [B, C, D]
front()         // returns B (no removal)
enqueue("E")    // [B, C, D, E]

Q5 · Binary Search Tree

BST Rules

core
  • Root = the first element inserted
  • For each node: left subtree contains values less than the node, right subtree contains values greater than the node
  • Insert one element at a time. Walk down: if smaller go left, if larger go right. Place at first empty spot.

2025 — BST from ["M","B","A","K","H","V","T"]

model answer

Step-by-step insertion

  1. M → root
  2. B < M → goes left of M
  3. A < M → left · < B → left of B
  4. K < M → left · > B → right of B
  5. H < M → left · > B → right of B (=K) · < K → left of K
  6. V > M → right of M
  7. T > M → right · < V → left of V

Final structure

            [M]
           /   \
         [B]   [V]
         / \   /
       [A] [K][T]
           /
         [H]

idxLeft ptrDataRight ptr

3—A—

6—T—
7—H—

Root value = M (1 mark).

Mark scheme: empty pointers can be written as —, NULL, -1, or simply blank. But each node MUST have left and right pointer cells — missing them = no marks.

BST Traversals (forecast — could be asked)

memorise all 3

Pre-order: Root → Left → Right

For our tree: M, B, A, K, H, V, T

In-order: Left → Root → Right (gives sorted order!)

For our tree: A, B, H, K, M, T, V

Post-order: Left → Right → Root

For our tree: A, H, K, B, T, V, M

// pseudocode for in-order
PROCEDURE inOrder(node)
    IF node = NULL THEN RETURN
    inOrder(node.left)
    OUTPUT(node.data)
    inOrder(node.right)
ENDPROCEDURE

2025 — Writing Rules (4+4 marks)

model answers

(c)(i) sibling(X, Y) — share the same parent

sibling(X, Y) :-
    parent_child(Z, X),    % Z is parent of X
    parent_child(Z, Y),    % Z is parent of Y too
    X \= Y.                % but X is not Y

Mark points: same Z parent for both · X ≠ Y · proper Prolog syntax with :- and . · two ANDs.

(c)(ii) family(X, Y) — X is in family with Y

family(X, Y) :-
    spouse(X, Y);          % married, OR
    parent_child(X, Y);    % X is parent of Y, OR
    parent_child(Y, X).    % Y is parent of X

Mark points: three OR cases (semicolons) · spouse OR parent both directions · proper syntax.

How to write Prolog rules (method)

technique
  1. Start with the head: relationName(X, Y) :-
  2. List all the conditions that must be true, separated by , (AND)
  3. If there are alternative definitions, use ; (OR)
  4. Use X \= Y when you need things to be different
  5. End the whole rule with one period

2025 (c) — Test Data & Errors

3 sub-parts

(c)(i) Normal data — 1 mark

Any text without 'z': "apple", "hello", "world".

(c)(ii) Erroneous data — 1 mark

Text containing 'z' (causes the bug), or non-letters: "buzz", "123", "hi!", "buZz".

(c)(iii) Error type when input is "buZz" — 2 marks

  • Type of error: Runtime error (also called execution error)
  • Justification: Out-of-range array access. When the program processes 'z', it finds z = alphabet[25], then tries alphabet[26] — which doesn't exist.

(c)(iv) Pseudocode fix — 3 marks

IF text[i] = alphabet[j] THEN
    IF text[i] = 'z' THEN             // 1 mark: handle z
        result ← result + alphabet[0]   // 1 mark: wrap to 'a'
    ELSE
        result ← result + alphabet[j+1]  // 1 mark: keep else case
    ENDIF
ENDIF

Expert Systems (12.5.1.2)

forecast

An expert system is an AI program that emulates a human expert's decision-making in a specific field.

Four components

  1. Knowledge base — facts and rules from human experts
  2. Inference engine — applies rules to facts to draw conclusions
  3. User interface — how user asks questions
  4. Explanation facility — explains why a conclusion was reached

Simple expert system in Prolog

% knowledge base — facts
symptom(flu, fever).
symptom(flu, cough).
symptom(cold, runny_nose).

% rule — diagnose by matching symptoms
diagnose(P, D) :-
    has_symptom(P, S),
    symptom(D, S).

% query: diagnose(jake, X).

Use cases: medical diagnosis · financial advice · troubleshooting · legal reasoning.

2D Arrays (11.5.2.2)

forecast

Only 1D arrays were tested in 2025 (Q1). 2D arrays could be tested in a similar style.

Common patterns

// declare a 3×4 grid
matrix ← DIM arr[3][4]

// traverse all cells (row-major)
FOR i ← 0 TO 2 DO
    FOR j ← 0 TO 3 DO
        OUTPUT(matrix[i][j])
    ENDFOR
ENDFOR

// sum all elements
total ← 0
FOR i ← 0 TO rows-1 DO
    FOR j ← 0 TO cols-1 DO
        total ← total + matrix[i][j]
    ENDFOR
ENDFOR

// find max
max ← matrix[0][0]
FOR i ← 0 TO rows-1 DO
    FOR j ← 0 TO cols-1 DO
        IF matrix[i][j] > max THEN
            max ← matrix[i][j]
        ENDIF
    ENDFOR
ENDFOR

Uses: spreadsheets, game boards (chess, tic-tac-toe), images, schedules.
