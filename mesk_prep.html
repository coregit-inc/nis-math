F-02 Expert Systems
12.5.1.2 · likely P3 +

An expert system is an AI program that emulates the decision-making of a human expert in a specific domain.

Four components

  1. Knowledge base — facts and rules from human experts
  2. Inference engine — applies rules to facts to derive conclusions
  3. User interface — how the user asks questions
  4. Explanation facility — explains why a conclusion was reached

Simple expert system in Prolog (medical diagnosis)

% knowledge base — facts
symptom(flu, fever).
symptom(flu, cough).
symptom(cold, cough).
symptom(cold, runny_nose).

% rule — if patient has all symptoms of D, diagnose D
diagnose(Patient, Disease) :-
    has_symptom(Patient, Symptom),
    symptom(Disease, Symptom).

% query: diagnose(jake, X).

Use cases

  • Medical diagnosis · financial advice · troubleshooting · legal reasoning

F-06 Fetch–Decode–Execute Cycle
12.3.2.5 · likely P1 +

The three stages

  1. Fetch
    • Address in PC (Program Counter) is copied to MAR (Memory Address Register)
    • Instruction at that address is loaded into MDR (Memory Data Register)
    • Instruction copied from MDR into CIR (Current Instruction Register)
    • PC is incremented to point to the next instruction
  2. Decode
    • Control Unit (CU) decodes the opcode in CIR
    • Determines the operation to perform and operands needed
  3. Execute
    • ALU or registers carry out the operation
    • Result stored back in a register or memory
Registers to know: PC, MAR, MDR, CIR, ACC (accumulator).

Stack vs Queue

Last In, First Out. Push/pop at the same end. Used for: function calls, undo, backtracking.

First In, First Out. Add at rear, remove from front. Used for: print spool, task scheduling, BFS.

F-16 Insertion Sort (companion to Bubble)
11.5.2.4 · likely P3 +

The syllabus says "sort by insert and bubble" — bubble was tested, insertion is overdue.

How it works

Build the sorted list one element at a time. For each element, shift larger elements right and insert it in its correct position.

FOR i ← 1 TO n-1 DO
    key ← arr[i]
    j ← i - 1
    WHILE j >= 0 AND arr[j] > key DO
        arr[j+1] ← arr[j]
        j ← j - 1
    ENDWHILE
    arr[j+1] ← key
ENDFOR
Complexity: O(n²) worst case · O(n) best case (already sorted) · stable sort.

F-21 Blockchain
11.1.2.6 · possible P1/P2 +

Blockchain is a distributed digital ledger where data is stored in blocks linked cryptographically.

How it works

  1. A new transaction is broadcast to the network
  2. Network nodes validate the transaction (consensus)
  3. Validated transactions are bundled into a block
  4. The block contains a hash of the previous block — creating a chain
  5. Adding the block to all copies of the ledger makes it nearly impossible to alter history

Key properties

  • Decentralised — no single point of control
  • Immutable — past records can't be changed without breaking the chain
  • Transparent — all transactions visible to participants

Uses

    Cryptocurrency (Bitcoin, Ethereum) · supply chain tracking · digital identity · smart contracts · medical records

11.1.3.5 · possible P1 +

02 Data Security & Integrity
Q2 · 6 marks +

Security vs Integrity

Data Security

Protection from unauthorised access. Concerned with who can reach the data.

Data Integrity

Validity & accuracy of data. Concerned with whether data is correct and uncorrupted.

Consequences of Missing Measures

  • No backup → permanent data loss after hardware failure or attack
  • No disk mirroring → downtime during recovery; failed drive can't be replaced while running
  • Mirroring caveat: a virus replicates to mirror; deleting on one disk deletes both

Encryption (2 marks: describe coding + purpose)

Encryption is the process of converting plaintext into a coded form (ciphertext) using an algorithm and a key, so that only authorised parties with the correct key can decrypt and read it. It secures data in transit (online transactions, email, file transfers) and at rest.

State two differences between data security and data integrity.
▸ Show answer
(1) Security protects data from unauthorised access; integrity ensures data validity. (2) Security prevents external threats; integrity prevents human/input errors.

04 Logic Gates & Boolean Algebra
Q4 · 12 marks +

Identifying gates from truth tables

Output table 1 (1,1,1,0) — the only zero is when both inputs are 1 → that's NAND.

Output table 2 (0,1,1,1) — only zero when both inputs are 0 → that's OR.

Simplification using De Morgan's Law

Q₁ = ¬((A + ¬B) · (C + ¬D))

// De Morgan: ¬(X·Y) = ¬X + ¬Y
   = ¬(A + ¬B) + ¬(C + ¬D)

// De Morgan again: ¬(X+Y) = ¬X · ¬Y
   = (¬A · ¬¬B) + (¬C · ¬¬D)

// Involution / double complement: ¬¬X = X
   = ¬A·B + ¬C·D
Laws used: De Morgan's Law + Involution (double complementation).

Drawing logic circuits

For Q₂ = ¬((A + ¬B) · C): NOT B → OR with A → AND with C → NOT. Always draw an output line from the final gate.

05 Data Flow Diagrams
Q5 · 9 marks +

Required parts of Level 1 DFD

  • Entities: customer, administrator
  • Processes (≥4): view product, log in, place an order, add a product
  • Data stores (3): products, customers/administrators, orders
  • Labelled data flows between them

DFD Rules — never violate these

  • An entity cannot pass data directly to another entity (must go through a process)
  • Data cannot move directly from an entity to a data store without processing
  • Data cannot move directly from a data store to an entity without processing
  • Data cannot move directly between two data stores
  • Processes must be named with verbs

Level 1 → Level 0 transformation

  1. Delete the data stores
  2. Combine all processes into one (the system)

03 Binary Search & Complexity
Q3 · 8 marks +

Completed pseudocode

INPUT name
low ← 0
index ← -1
high ← LENGTH(PhoneBook) - 1
flag ← FALSE
WHILE (low <= high AND flag <> TRUE)
    mid ← INT((low+high)/2)
    IF PhoneBook[mid] = name THEN
        index ← mid
        flag  ← TRUE
    ELSE IF PhoneBook[mid] > name THEN
        high ← mid - 1
    ELSE
        low  ← mid + 1
    ENDIF
ENDWHILE
IF flag = TRUE THEN
    OUTPUT("Found at index:", index)
ELSE
    OUTPUT("Not in array")
ENDIF

Complexity

  • Time complexity: O(log n)
  • For n = 100, worst case = 7 iterations (because ⌈log₂ 100⌉ = 7)

06 Prolog · Facts, Rules, Queries
Q6 · 11 marks +

Facts vs Rules

  • Facts: lines 1–10 (parent_child & spouse with concrete names)
  • Rules: lines 11–12 (use :-)

Query: child(jiger, Y)

The rule child(X, Y) :- parent_child(Y, X) means Y is the parent of X. Find Y where parent_child(Y, jiger). From line 7: parent_child(bolat, jiger). So Y = bolat.

siblings(X, Y) — share a parent

sibling(X, Y) :-
    parent_child(Z, X),
    parent_child(Z, Y),
    X \= Y.

family(X, Y) — X and Y are family

family(X, Y) :-
    spouse(X, Y);
    parent_child(X, Y);
    parent_child(Y, X).
Symbols: comma , = AND · semicolon ; = OR · \= = not equal.

Caesar Encrypt — error analysis

Normal data: any text without 'z' (e.g., "apple")
Erroneous: text containing 'z', or numbers/symbols (e.g., "buZz")

Error type: Runtime / execution error — out-of-range array access. When z matches alphabet[25], the code tries alphabet[26] which doesn't exist.

Fix

IF text[i] = alphabet[j] THEN
    IF text[i] = 'z' THEN
        result ← result + alphabet[0]
    ELSE
        result ← result + alphabet[j+1]
    ENDIF
ENDIF
